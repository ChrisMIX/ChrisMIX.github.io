We're going off the Rails on this Ruby train...

Hello again. Today I'm here to tell you all about Rails. What is Rails you may ask? Rails is an extension of the Ruby language we already know and love. It is also known as Ruby on Rails, just in case you weren't aware already. You install it using the command line on your computer, and it allows you to build websites using the Ruby language. Rails takes website building and streamlines the process. It combines Ruby with the likes of HTML, CSS, and Javascript. and runs on a web server. For this reason, Rails is considered a "back-end" web development platform (a browser would be considered the "front-end"). Now why would you want to use Rails? Allow me to explain.

Rails is a website building tool that is built on a Ruby framework. What is great about this is that it comes with the benefits of Ruby. Also, buliding a web app in Rails is significantly quicker than it would be in a java framework. The other thing worth noting is that Rails is full stack, meaning that every piece of the web app works together seamlessly and it takes less code than with other altervatives. It also means that you have everything you need to build a database driven web app. Rails also gives you three environments to use (development, testing, and production) that help the development process overall. Now the gist of what I'm saying here is that Rails streamlines the web development process, and is built on a Ruby framework. It takes older conventions from other frameworks and changes them in favor of an exprience that is much more intuitive and user friendly. Rails aims to be the whole web development package in one place. Now there are more nitty gritty things to be found regarding Rails, but this is all I have to say for now. Thanks for reading!
