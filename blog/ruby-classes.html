Let's Get Classy!

11/4/15

Welcome back to my blog! Today we are talking about something called a class. What can it do? Plenty of cool stuff. Take a look around the room you're in right now. Pick any object you find. In my case, I am typing this up on a laptop, so I'll use that for this explanation. Now what makes my laptop a laptop? You might be thinking of a number of things, like the size, portability, design, or a number of other things. By themselves, these terms and others simply are words. Together, they describe a laptop. That's in a way what a class is all about. They are containers that hold attributes and methods together under one roof. What's inside is individual pieces of what defines that class. Now this may not be making sense yet, so let's go to an example. Check out this code!

class Laptop
def initialize(serial_number, screen_size)

@serial_number = serial_number
@screen_size = screen_size
end
end

Now quite a few things are going on here, so allow me to explain. A class definition is started by writing class and then the class name capitalized. The initialize method that comes next tells us what the laptop class needs as input. In this case, we have a serial number, and a screen size. That all seems fairly simple, but why the @ symbol? That signifies that these variables are instance variables. Instance variables have a greater scope than your average local variable. Normally variables are only available to use in the context they are created in. With instance variables, they are available for use by any method in that class. This is very helpful when you have data that you want to be used by multple methods. Now something else we need to talk about are instance methods, but first let's look at the next batch of code...

class Laptop
def initialize(serial_number, screen_size)

@serial_number = serial_number
@screen_size = screen_size
end

def get_serial_number
return @serial_number
end
end

macbook = Laptop.new("1234", "13 inches")

So here we implemented two more things. Notice the new method here called get_serial_number. That is an instance method. What that means is that when you create an instance of the Laptop class, it inherently has that method built in. That means if I call get_serial_number on macbook below, I would get "1234" as a return value. Aside from that, defining a class also is fairly straightforward. You simply type classname.new(parameters). Well that more or less covers classes for now. Until next time!
