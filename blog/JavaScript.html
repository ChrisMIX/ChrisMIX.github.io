Have a Nice Cup of JavaScript

11/9/15

Welcome back! Today I am here to tell you all about Looping in Ruby and Javascript. At their core, they both accomplish the same tasks. The syntax can differ a bit between the two languages however. Let's consider an example. Imagine we want to write a program that prints phrase "Cool story bro." ten times. We will start by using the while loop for both programs. Check out the following codeblock...

#Ruby first
counter = 1
while counter <= 10
puts "Cool story bro."
counter += 1
end

//Javascript Next
var counter = 1 var statement = "Cool story bro."
while (counter <= 10){
console.log("Cool story bro.")
counter++
}

Now both of these will do the same exact things, but they have different ways of doing it. With Javascript, we first establish our counter. Then we establish our condition that will keep printing the statement we want. We increment our counter and then keep going until the counter hits 11. Ultimately what I described is true for Ruby as well, but the syntax is slightly different. For loops work similarly in both Ruby and Javascript as well. Take a look...

#Ruby
for num in 1..10
puts "Cool story bro."
end

//Javascript
for (var counter = 0; counter <= 10; counter++)
console.log("Cool story bro.");

Now both of these loops are very shortened versions of the while loops we saw a moment ago. That's why the for loop is used. Javascript manages to take the already short for loop from Ruby and take one line away on top of it. Javascript's for loop has three different parameters in parentheses telling the loop what to do. The first defines your counter. The second tells the loop when to stop and the third increments the counter. Aside from that, we just use a print statement. Now you see how the looping process of Javascript and Ruby really isn't that different. Well that's it for now. Tune in again next week!
