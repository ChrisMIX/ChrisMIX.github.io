The Incredible Enumerable Methodical Map

10/29/15

Hey all. This time we are here to talk about a very important enumerable method called map. Now there isn't too much to say about this one, but it is a very powerful tool. As always, I will provide an example to help nail down the idea.

Alright so to start this explanation it is important to note that map is a method that primarily works with arrays. The way map works is that you can pass a codeblock to it as well as an array, and it will apply the codeblock to each array element. Consider the following example: I have a list of numbers and I want each one to be multiplied by 10. This is a fairly simple request with map. Take a look at this block of code...

array = [1, 2, 3, 4, 5]
array.map {|x| x * 10}

It is important to note that you can also use do and end instead of the curly brackets. Now you might be thinking, "How is this any different from using each"? Well, I will tell you just that. If you tried using each for this task, your return would be...

[1, 2, 3, 4, 5]

Your return from using map would be...

[10, 20, 30, 40, 50]

Notice that each doesn't even give us what we asked for. That's because each does the task we ask for, but it doesn't chaneg the original array. Map takes our directions and returns to us the brand new array. Each is used more for printing things rather than changing anything.

Now that pretty much covers the explanation on map, but before we finish I want to leave a couple more interesting examples to illustrate the cool things you can do with this method.

Let's consider two more examples. In the first, we will write a map statement that takes each element of the array and deletes each element individually. In the second method we will add 1 to each array element. The code for those map statements would look like this...

array = [1, 2, 3, 4, 5]
array.map {|x| x.delete}

array = [1, 2, 3, 4, 5]
array.map {|x| x + 1}

The results from these would be an empty array, and the array [2, 3, 4, 5, 6]. Now that pretty much covers the discussion on map. If you have any questions, feel free to contact me.
