Ruby Hashes, Arrays, Stings, Integers

Hashes

EX: {"key" => "value"}
-Can store multiple things in a |key, value| style.
-Keys can be any data type you want (String, Symbol, Integer, etc.)
-Values can be any data type as well, even another hash!
-You can iterate over a hash using hash.each {|key, value| codeblock}
-Very versatile when compared with arrays. Great if you want a value to have one specific key.

Arrays

EX: [1, 2, 3, 4, 5]
-Similar to Hashes, but keys are static.
-Values can be moved around and the keys stay the same.
-Values can be any data type, even an array or a hash!
-Can be iterated over using array.each {|value| codeblock}
Great if you want a more simplistic data type that has less complicated syntax.

Strings

EX: "Ruby Review"
-Strings are a data type contained in quotes.
-They are great for giving input to the user and giving feedback to someone running the code.
-Strings have many built in methods like length, reverse, capitalize, upcase, downcase (These do what you would think)
-You can combine Strings using +

Integers

Ex: 42
-Integers are whole numbers without decimal points
-Standard mathematical operations built in (+, -, *, /)
-Integers are the bread and butter of any language.
-If you want decimal points, use floats instead!
